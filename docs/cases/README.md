# docs/cases/

This directory is intended to hold **motivational case documents** — one per active case.
These are ARW-layer documents (not ART pipeline artifacts) that explain *why* a given system
was selected as an ARW test case and what it contributes to the research program.

## Intended structure (per case)

Each file should cover:
- System and its relevance to ARW/ART
- Which BC class(es) it exemplifies and why this is non-trivial
- What the case adds that no existing case already provides
- Links to the corresponding `cases/CASE-YYYYMMDD-####/` pipeline artifacts
