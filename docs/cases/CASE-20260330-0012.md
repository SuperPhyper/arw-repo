# Case Motivation: CASE-20260330-0012
## Labyrinth Agent — Emergent Modes Experiment

**Case ID:** CASE-20260330-0012
**System:** Context-navigation labyrinth agent (unstructured RL policy)
**Primary BC class:** Restriction (primary), Coupling (co-driver)
**Pipeline artifacts:** [cases/CASE-20260330-0012/](../../cases/CASE-20260330-0012/)

---

## Why this system?

This case asks a direct question about the scope of the ARW regime framework:

> Is BC-induced regime structure a property of physical dynamical systems only,
> or can it arise in any learning system exposed to structured boundary conditions?

All prior ARW cases share a structural feature: the system's regime structure
is either analytically known (Kuramoto, Doppelpendel) or imposed by architectural
design (CASE-20260329-0011 with its prescribed mode library). In none of these
cases does the ARW framework have to find regime structure that was neither
analytically derived nor architecturally prescribed.

This case removes both scaffolds. The agent is a single unstructured policy.
No one prescribed its modes. ARW is applied purely as an observation instrument —
asking whether the training dynamics, confronted with a structured environment,
produced regime-like behavioral differentiation on their own.

A positive result means ARW can detect emergent regime structure in a domain
where no ground truth exists. A negative result means the framework's detection
capability is limited to systems with explicit structural support.

---

## BC Class: Restriction (primary), Coupling (co-driver)

**Primary BC class: Restriction**

The constitutive boundary condition of S_emergent is a *negative* Restriction:
the agent is what it is **not given**. No mode library, no mode selector, no
mode-labeled reward. This restriction on architectural provision is what makes
the experiment meaningful — if any mode structure is found, it must be emergent.

Zone R environments (tight resource/action constraints) instantiate positive
Restriction BCs, potentially inducing conservative behavioral strategies.

**Co-driver: Coupling**

The agent can only observe zone features locally (visibility radius r=5). This
perceptual coupling is the only channel through which zone structure reaches the
agent's policy. Without it, zone type could not induce behavioral differentiation.
The coupling is what makes emergence *possible*; the Restriction is what makes
it *non-trivial*.

**What is non-trivial about this BC classification:**
The negative Restriction (absence of architectural modes) is a BC class in the
same formal sense as positive Restrictions (active constraints) — it defines
a boundary on the admissible state space (the policy's behavioral space). But its
falsification logic is inverted: a positive Restriction produces a smaller
admissible set by exclusion; a negative Restriction produces a *larger* admissible
set by permission. The expected regime structure, if any, must arise from task
optimization within this unconstrained policy space, mediated by the Coupling BC.

## What this case adds

| Prior gap | This case's contribution |
|-----------|--------------------------|
| All cases have known or prescribed regime structure | First case where regime structure must be discovered, not confirmed |
| ARW as design tool or validator | ARW as pure observation instrument: no design intent involved |
| No precedent for negative-Restriction BC | First BC formulation defined by architectural absence |
| CASE-0004 established physical emergence (Stuart-Landau) | This case extends emergence detection to a behavioral/cognitive system |
| No cross-paradigm comparison (designed vs. emergent) | Together with CASE-0011: first paired designed/emergent comparison on the same environment |

**Theoretical significance:**
If H1 is confirmed (stable ε-plateau at N = zone count), it implies that
ARW regime structure is a property of the *environment's BC structure*, not of
the agent's architecture. The environment induces regime-like differentiation
even in an unstructured learner. This would strengthen the ARW claim from
"regime structure arises in systems with explicit BC design" to "regime structure
arises in any adaptive system exposed to structured boundary conditions."

If H1 is falsified (no stable partition emerges), it implies that architectural
scaffolding (mode library, consolidation, explicit regime support) is a necessary
condition for BC-structured regime formation — a meaningful negative result that
constrains the generality of the framework.

## Complement experiment

CASE-20260329-0011 (Designed Modes) and CASE-20260330-0012 (Emergent Modes) are
complementary. CASE-0011 tests whether prescribed architectural modes produce a
measurable ARW partition. This case tests whether such a partition arises *without*
prescription. Together they bracket the role of architectural design in regime formation.
