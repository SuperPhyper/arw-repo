# Between the sessions
### What the slow loop does — three passes and a handoff

*The companion to the in-room moves. The guide keeps sending work "between sessions" but
never lays out the work. This is it. Governing rule: the slow loop is the **only** place
contested-scope work is allowed to happen — that's what keeps the live room from
polarizing.*

---

## The job of the slow loop

The between-session phase has one job: **turn what happened in the room into what opens
the next room.** Not "prep an agenda" — decide what kind of work the next session must
do, do the part that can't be done live, and hand back a clarified frame.

It runs on one discipline: **loop separation.** Live work and frame work never run at
once. The room does calibration; the slow loop does everything that would polarize a
room. The output of the slow loop is always a *proposal or a clarified frame* — never a
decision made for the group in their absence.

---

## The common spine (every pass)

Run these four either side of the branch:

1. **Read the session you just ran — the kind, not the content.** Did it stay Tuning, did
   the frame come apart, or did you hit the wall? Use the same live-read signals. This
   read routes the whole pass.
2. **Capture raw, before you interpret.** Internal working notes: each party's question
   and hard constraint *as actually voiced*, what went quiet, where tension didn't drop
   when you "settled" something. Don't smooth it yet.
3. **→ Branch** on the read (A, B, or C below).
4. **Verify against representation.** Whatever the branch produces, test it before it goes
   back: is every party still represented, and still locally negotiable? If a party has
   gone unrepresentable, the pass isn't finished.
5. **Hand off.** Return to the group with the branch's product *plus* a short recap. Every
   session opens with this handoff — that's the hinge that connects the loops without
   collapsing them.

---

## Branch A — It stayed Tuning

The light pass. Calibration is working; keep it clean.

1. Synthesize where the parameters landed — what converged, what's still open.
2. Re-surface anything that went quiet, so no constraint quietly drops out.
3. Prep the next agenda as continued convergence: the open dials, in priority order.
4. **Handoff:** a short "here's where we are / here's the next dial" proposal.

## Branch B — It became Framing

The heavy pass. This is where the friction work runs (see
[Crystallize the friction](facilitation_crystallize_the_friction.md)).

1. **Surface the native questions** captured in step 2 — each party's own goal and
   measure, in its own words.
2. **Run the crystallization step:** collide the questions (don't cluster), name each
   friction so both sides own it, diagnose each (missing piece / uneven leverage / no
   overlap), and draft the local compromise where one exists.
3. **Reconstruct the frame:** assemble the named frictions into a *candidate shared
   question* each party can recognize itself in.
4. **Test the candidate off the clock.** Use a between-session survey to run the
   recognition test with each party individually — *before* bringing it live. Recognition
   that would polarize in the room can happen quietly here.
5. Representation gate (spine step 4).
6. **Handoff:** bring back the clarified question *and* the named frictions — not a
   solution. The next session re-opens as Tuning on the new frame.

## Branch C — It hit the wall

The different pass. The frame is fine; don't reframe. The problem is who moves first.

1. **Don't crystallize more frictions** — that's the wrong axis here. Diagnose the
   first-mover structure instead: what makes each party safer not moving?
2. **Redesign the path:** a smaller first move, reciprocal or simultaneous moves,
   reversible trials — lower how much will each step demands.
3. **If no first-move structure changes it:** prepare to name the limit to the sponsor —
   the honest stop. Do **not** schedule another productive-looking session that changes
   nothing.
4. **Handoff:** either a lowered-barrier next step, or a straight conversation with the
   sponsor and group about the will — named, not buried.

---

## Cadence within a pass

A reliable order for the work itself:

**reflective re-working** (sit with the raw capture) → **synthesis** (the facilitation
team builds the proposal or frame) → **check against participants** (between-session
survey: does the candidate land?) → **return with a proposal** (the handoff).

Keep the loop separation honest inside the pass too: the synthesis produces a *proposal*,
not a verdict. The moment the slow loop starts quietly deciding *for* the group, it has
stopped being the slow loop and become an unaccountable back room.

---

*Where this connects: Branch B is the home of the friction/recognition work; the
representation gate is the participation incentive; Branch C's path redesign and honest
stop are the will-limit from the guide's Part II. The handoff is what makes "each session
opens with a clarified frame" actually happen.*
