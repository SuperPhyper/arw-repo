# Repository Weak Points --- Updated Assessment

This document tracks structural gaps and underdeveloped parts of the
repository.

It has been updated after a full repo audit to distinguish between:

-   **resolved items** (already implemented)
-   **partially resolved items** (work started but incomplete)
-   **open gaps** (still missing)

The goal is to prevent chasing problems that are already solved.

------------------------------------------------------------------------

# Category 1 --- Critical Structural Components

These components are foundational to the ARW research program.

## BC Taxonomy

File\
`docs/bc_taxonomy/boundary_condition_classes.md`

Status\
**Open --- still underdeveloped**

Current state - BC classes listed (restriction, coupling, symmetry
breaking, dissipation, forcing, aggregation) - No formal definitions -
No regime consequences - No example mapping

Missing - formal definition of each BC class - regime structures
generated by each class - cross-system examples - interaction rules
between BC classes

Importance This taxonomy is central to the claim:

> boundary-condition classes generate characteristic regime partitions.

Without definitions this claim cannot be tested.

Priority: **high**

# Overall Diagnosis

Most weak points are no longer **missing ideas**.

They are **canonicalization tasks**:

-   consolidating definitions
-   formalizing metrics
-   organizing reference files

The framework itself is already largely present across the repository.
The remaining work is primarily **systematization** rather than
invention.
