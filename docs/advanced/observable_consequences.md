# Consequences of Observable Decomposition

## Context

This document draws consequences from the full observable decomposition
analysis (r_ss, var_rel, lambda_proxy, σ²(θ)). The central finding is
that all four observables share a common dynamic exclusion zone Z_shared,
generated by assumptions about stationary measures and ergodicity.

Prerequisites:
- `docs/advanced/observable_decomposition.md`
- `docs/advanced/latent_degrees_of_freedom.md`
- `docs/glossary/observable_range.md`

---

## K1: Z_shared is Enforced by System Dynamics

Z_shared is not an observable-specific exclusion zone.
It is a **lower bound** for all observables of a class:

```
Definition of observable class E:
    π ∈ E  ↔  substrate of π contains the assumption
              "μ_stat exists, is unique, ergodicity holds"

Theorem:
    For all π ∈ E:  Z(π) ⊇ Z_shared

    Z_shared = { p ∈ P | ergodicity violated
                      OR μ_stat not unique
                      OR critical slowing down diverges }
```

For the Kuramoto phase transition, κ_c ∈ Z_shared holds by construction:
ergodicity breaks at κ_c, μ_stat is not unique at κ_c,
and critical slowing down diverges at κ_c.

**Consequence:** No scope with π ∈ E can have κ_c as an interior point
of its observable range. This is not a measurement problem —
it is a structural constraint of the observable class.

---

## K2: Scopes Measure Doubly Projected Physics

The implicit assumption in ARW has been:

```
Scope → Observable → Regime partition → Physics
```

The decomposition analysis reveals the actual structure:

```
P(θ, t)                         full physics
    ↓  pre-scope projection      [substrate of π]
π(x)                            observable
    ↓  ε-clustering
P_S                             regime partition
    ↓  interpretation
Physical regimes ?
```

Both projections discard information. The regime labels are
**doubly projected physics** — projections of projections from P(θ, t).

**Consequence:** The completeness of the regime partition P_S relative
to the physical regime structure is not guaranteed. P_S is a partition
of observable values, not of physical states. Whether P_S = R_true
(the true partition based on P(θ, t)) holds is an open empirical
question — see Q_NEW_6.

## K3: Phase Transition as Scope Transition, Not Regime Boundary

This is the conceptually sharpest consequence.

```
Regime boundary θ*:   observable changes discretely within R(π)
                      — scope remains valid

Scope transition:     observable leaves R(π)
                      — different π or different S needed
```

Since κ_c ∈ Z_shared ⊆ Z(r_ss), the phase transition lies outside
the observable range of r_ss. The transition observed in CASE-0001
at θ* ≈ 1.475 is strictly speaking a **scope transition**,
not a regime boundary in the ARW sense.

```
S_incoherent = (B_inc, {r_ss}, Δ, ε)    valid for κ ≪ κ_c
S_coherent   = (B_coh, {r_ss}, Δ, ε)    valid for κ ≫ κ_c
κ_c ∉ R(r_ss)  →  κ_c is the boundary between S_incoherent and S_coherent,
                   not an interior point of either scope
```

**Implication for CASE-0001:** The go_nogo status and regime partition
remain valid — but the interpretation of θ* as "phase transition" should
be more precisely framed as "scope transition at observable collapse."

**Implication for ARW generally:** The framework needs a formal distinction
between regime boundary (within a scope) and scope transition (boundary
between scopes). This distinction is not yet explicitly encoded.

## K4: Transfer Metrics Measure Observable Transfer, Not System Transfer

Transfer analysis between two cases yields Φ as a measure of
transferability. However:

```
Φ = f(S_A, S_B)  —  function of the scopes
                 —  not a function of the systems
```

Two systems with identical physical BC structure but different observables
would yield low transfer — not because the systems differ, but because
the observables make different projections from P(θ, t).

For CASE-0001 ↔ CASE-0003 (Φ=0.40 raw):

```
CASE-0001: system BC = Coupling     observable BC = R³·A·D
CASE-0003: system BC = Restriction  observable BC = R³·A²·S

The observables project away the system BCs.
What transfers is the observable BC structure —
both Restriction-dominated, but with different
aggregation and scaling sub-structure.
```

**Consequence:** Φ=0.40 could mean either:
- (a) The systems are physically different — or
- (b) The observables project the shared structure differently

Without knowledge of the observable BC structures, Φ is not
unambiguously interpretable as system transfer.

**Proposal:** Transfer reports should additionally document the
observable BC structures of both scopes, making (a) and (b) separable.

## K5: BC Class — System Property or Scope Property?

The decomposition analysis shows that BC classes appear in the observable
itself — independently of the system. r_ss is structurally
Restriction-dominated regardless of which system it observes.

This raises a foundational question for the BC taxonomy program:

```
Position A (System-BC):
    BC class describes the system.
    The observable must be chosen such that its BC structure
    resonates with the system BC.
    Mismatch → poor transfer, fragile partition.

Position B (Scope-BC):
    BC class describes the scope (system + observable).
    Different observables of the same system can induce
    different BC classes.
    BC class is observer-dependent.
```

Position A is the more conservative reading and consistent with current
ARW practice (BC class in BCManifest = system property).

Position B has far-reaching consequences: BCManifest would need to carry
both system BC and observable BC. Transfer would only be meaningfully
interpretable between compatible observable BC structures.

**Status:** open foundational question — see Q_NEW_9.

## K6: Fluctuation Observables as Structural Solution for Z_shared

If stationary expectations structurally fail at phase transitions,
which observable class would not fail?

```
Required: π ∉ E  (without assumption "μ_stat unique and ergodic")
```

Four candidate classes:

```
(a) Finite-time statistics:  π(T) without T → ∞ limit
    + carries transient information
    + no stationary measure needed
    - T-dependent, difficult to compare across cases

(b) Distribution observables:  π = P_T(θ)
    + carries multimodality, asymmetry
    + no expectation collapse needed
    - high-dimensional, ε-clustering more demanding

(c) Topological observables:  π = w_topo, Betti numbers
    + discrete, robust against dynamical details
    + no stationary measure assumed
    - domain-specific, difficult to generalize

(d) Fluctuation observables:  χ = ∂r_ss/∂κ (susceptibility)
    + diverges at κ_c → maximum signal there
    + uses non-stationarity as signal, not as noise
    - requires dense κ-sampling, not computable from single simulation
```

Class (d) is particularly relevant for ARW:

```
χ = ∂r_ss/∂κ

χ ∈ Z(r_ss)?  No — χ has no stationary expectation as a substrate
              assumption. χ is a derivative, not a mean.

R(χ) contains κ_c?  Yes — χ diverges at κ_c.
                    This is maximum sensitivity, not collapse.
```

**Consequence for CASE-0001:** χ is a candidate for a new observable
specifically designed for the scope transition at κ_c. It would not
measure the regimes far from κ_c (r_ss is better there), but the
transition itself.
