# ARW Hypothesis Structure Diagram

This document summarizes the **current structural hypothesis stack**
emerging from the ARW operator program.

It provides a condensed structural map connecting:

- boundary-condition classes
- operator signatures
- primitive operator bases
- scope formalization
- mathematical scope limits

The diagram should be read as a **research structure**, not as a final theory.

# 2. Condensed Structural Chain

The chain can be read in two directions with different semantics.

```
ANALYTICAL DIRECTION                    GENERATIVE DIRECTION
(observation → structure)               (structure → phenomenon)

Observed system structure               Primitive operator basis
        ↓                                       ↓
Boundary conditions          ←→         Coupling dimensions
        ↓                               (time / parameter / stochastic)
Operator signatures                             ↓
        ↓                               Operator chains / BC classes
Primitive operator basis                        ↓
        ↓                               Flow structures
Coupling dimensions                             ↓
        ↓                               Regime partitions
Flow structures                                 ↓
        ↓                               Empirical scope S = (B, Π, Δ, ε)
Regime partitions                               ↓
        ↓                               Mathematical scope boundary
Empirical scope S = (B, Π, Δ, ε)
        ↓
Mathematical scope boundary
```

**Analytical direction:** Starting from observation, one asks which BC classes
are active, which operator signatures they produce, and which primitives underlie them.
This is the direction of ARW case analysis.

**Generative direction:** Starting from the primitive operator basis, one asks
which operator chains are possible, which BC classes they produce, and which
regime partitions emerge. This is the direction of ARW scope construction.

**Q15 — Bidirectionality:** The question whether these two directions are
mutually invertible is the central open question of the operator program.
The Littman–Metcalf case (`docs/art_instantiations/arw_littman_bc_analysis.md`)
provides the first partial positive answer: within a single scope with
non-overlapping operator signatures, the inverse inference from operator
structure to BC classes is unambiguous.

The general condition for invertibility remains open.

# 4. Interpretation

The diagram proposes that regime theory can be understood as the
interaction of:

- boundary conditions
- operator compositions
- coupling dimensions
- flow geometry

If this hypothesis holds, ARW becomes a **structural theory of regime
formation**, while ART provides the quantitative mathematical tools
needed for concrete system models.

# 5. Status

This diagram reflects the **current hypothesis state of the operator program**
and is intended as a reference structure for ongoing development.

Each level carries a different epistemic status:

| Level | Content | Status |
|---|---|---|
| [0] Empirical start | observed phenomena | empirical fact |
| [1] BC level | static/dynamic stratification | working-hypothesis |
| [2] Operator signatures | BC → operator families | working-definition |
| [3] Primitive basis | {composition, product, projection} | working-hypothesis |
| [4] Coupling dimensions | time/parameter/stochastic | working-hypothesis |
| [5] Flow/limit level | asymptotic projection | hypothesis |
| [6] Scope level | ε tension, structural components | working-definition |
| [7] Mathematical scope | F5, closure interpretation | hypothesis |
| [8] Research program | Q-OB1–Q-OB5 | open-question |

The weakest links in the chain — where the hypothesis could most easily fail —
are [3]→[4] (whether time coupling is derivable from product + composition)
and [5] (whether asymptotic projection is structurally equivalent to
finite projection for all BC purposes).

# 6. Bidirectional Readings and Invertibility

The stack can be traversed in two directions, corresponding to two
distinct research activities:

**Top-down (generative):** Given a primitive operator basis, derive
BC classes, then operator signatures, then scope structure, then
regime partitions. This is scope *construction* — the forward direction
of ARW formalization.

**Bottom-up (analytical):** Given observed regime structure, infer
operator signatures, then BC classes, then primitive basis. This is
scope *diagnosis* — the inverse direction, relevant for applying ARW
to new domains.

The bidirectionality question (Q15) asks whether the bottom-up direction
is well-defined — whether the map from operator signatures to BC classes
is injective (one-to-one) within a given scope.

The current evidence suggests:

| Condition | Invertibility |
|---|---|
| Non-overlapping operator signatures | unambiguous — Littman case |
| Single dominant BC class | unambiguous |
| Multiple overlapping BCs | ambiguous — social scopes |
| Scope transition underway | fails — operator vocabulary shifts |

The F5 failure mode (mathematical scope exhaustion) represents the
extreme case where bottom-up inference is impossible not because of
BC overlap, but because the observable Π is constructed in an
insufficient operator language — the analytical direction has no
starting point.
