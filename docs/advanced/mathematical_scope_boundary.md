# Mathematical Scope Boundaries and Primitive Operator Bases

## Overview

Within the ARW framework, boundary-condition (BC) classes are interpreted as
structures generated by **compositions of primitive operators** acting on
system degrees of freedom, potentially coupled with variables such as time
or parameters.

This document explores a key implication of this view:

> The limits of a mathematical scope correspond to the limits of the
> available **primitive operator basis** used to formalize system structure.

If a structural operation observed in a system cannot be represented as a
composition of available primitive operators, then the **mathematical scope
of the current formalization has been reached**.

---

# 1. Mathematical Formalization as Operator Composition

In this framework, mathematical models arise from:

primitive operators  
→ operator compositions  
→ structural relations  
→ dynamical equations  
→ regime structures

Primitive operators form the **structural alphabet** of the formal system.

Examples include:

| Primitive operator | Structural interpretation |
|---|---|
| Addition | superposition of contributions |
| Multiplication | scaling or interaction |
| Projection | restriction of admissible states |
| Product construction | extension of state space |
| Composition | dependency chains between variables |

More complex BC structures emerge from compositions of these primitives.

---

# 2. Boundary Conditions as Operator Structures

BC classes can be interpreted as **families of operator constructions**.

Examples:

Restriction  
projection operator

Coupling  
multiplicative or compositional interaction

Forcing  
time ⊗ (multiplication + addition)

Dissipation  
time ⊗ scaling

These constructions generate vector fields and flow geometries that
produce observable regime structures.

---

# 3. Mathematical Scope

A **mathematical scope** can therefore be defined as:

> The set of structural relations that can be represented using a given
> primitive operator basis and its admissible compositions.

Formally:

Mathematical scope = closure of primitive operators under composition.

Any model built within this scope can only represent structures belonging
to this closure.

---

# 4. Detecting Scope Boundaries

A boundary of the mathematical scope is reached when an observed structural
operation cannot be expressed as a composition of the available primitive
operators.

In such a situation one of three conclusions must follow:

1. The model is incomplete or incorrectly structured.
2. The primitive operator basis is incomplete.
3. The current mathematical formalism is insufficient to capture the structure.

Historically, such situations have often led to extensions of mathematics.

Examples include:

| Problem | Extension introduced |
|---|---|
Negative results in subtraction | negative numbers |
Polynomial roots | complex numbers |
Non-smooth distributions | generalized functions |
Quantum observables | non‑commutative operator algebras |

In each case the observed structure required **expanding the operator basis**.

---

# 5. ARW Interpretation

Within ARW this idea can be expressed as:

BC structures  
→ operator constructions  
→ regime formation

If a BC structure cannot be represented within the current operator basis,
then the **mathematical scope of the framework has been exceeded**.

This does not necessarily imply that mathematics itself has reached a final
limit. Rather, it indicates that the **current operator basis is insufficient**
for representing the observed structure.

---

# 5a. Bridge: Mathematical Scope and Empirical Scope

The mathematical scope defined in Section 3 — the closure of primitive operators
under composition — is a formal construct.

The ARW empirical scope S = (B, Π, Δ, ε) is an observational construct.

The relationship between them is not trivial, and it must be made explicit.

**Claim:** The mathematical scope of a formalization constrains the empirical
scope it can support.

Specifically:

- **B** (boundary constraints selecting X_B ⊆ X) requires that the admissibility
  condition be expressible in the operator language. If the physical boundary
  mechanism involves a structural operation outside the current operator basis,
  then X_B cannot be correctly specified — the model includes inadmissible states
  or excludes admissible ones.

- **Π** (admissible descriptions / projections) requires that the observable
  map π: X → D be constructible from the available operators. If the relevant
  observational structure cannot be expressed as an operator composition,
  Π is incomplete — the partition is unresolvable not because ε is too large,
  but because the observable itself is inexpressible.

- **Δ** (admissible perturbations) requires that perturbation directions be
  definable within the operator language. If a physically relevant perturbation
  has no representation in the current basis, it is invisible to the formal
  scope — the robustness test is structurally incomplete.

- **ε** (resolution threshold) presupposes that d_Π(x, y) is computable.
  If Π is inexpressible, ε becomes formally undefined for the missing structure.

**Consequence for scope failure:**

When the mathematical scope is exceeded, the empirical scope does not simply
become less precise — it becomes **structurally malformed** in one or more
of its four components.

This produces a characteristic failure signature distinguishable from
ordinary observable insufficiency:

| Failure type | Empirical symptom | Structural cause |
|---|---|---|
| B malformed | inadmissible states populate the data | boundary mechanism inexpressible |
| Π incomplete | no observable resolves the partition | observable map inexpressible |
| Δ incomplete | robustness test misses relevant perturbations | perturbation inexpressible |
| ε undefined | resolution threshold cannot be set consistently | distance metric inexpressible |

In the ARW falsification framework, mathematical scope exhaustion most
directly manifests as **F3 (collective observable failure)**: all observables
in Π fail simultaneously, not because individual observables are insufficient
(which would be observable insufficiency, not scope rejection), but because
the entire Π is constructed in an operator language that cannot represent
the relevant structure.

This is a new failure mode not fully captured by F1–F4 as currently defined.
It could be designated **F5: mathematical scope exhaustion** —
collective Π failure caused by operator basis insufficiency rather than
by partition non-existence or boundary artifacts.

---

# 6. Methodological Consequences

This perspective reframes several questions in the ARW research program.

Instead of asking:

Are the BC classes complete?

one asks:

Is the primitive operator basis sufficient to generate all BC classes?

Completeness of the BC taxonomy becomes a **derivation problem**
within the operator algebra.

---

# 7. Minimal Operator Bases

If BC classes emerge from operator compositions, an important theoretical
goal becomes identifying the **minimal primitive operator basis** capable of
generating the full set of BC structures.

Such a basis would function as a structural foundation for:

- BC taxonomy
- regime classification
- dynamical model construction

### Candidate minimal basis

Based on the BC classes established so far, the following set of primitive
operators appears necessary:

| Primitive | Required for | Notes |
|---|---|---|
| Addition | Forcing, Aggregation | superposition of independent contributions |
| Multiplication / scaling | Dissipation, Forcing, Coupling | proportional interaction and decay |
| Projection | Restriction, Symmetry breaking | selection of admissible subspaces |
| Product construction | Coupling, state-space extension | joining independent subsystems |
| Composition | all BC classes | dependency chaining between variables |

**Time coupling** is not a primitive operator in this list — it is a
*coupling of composition with a distinguished degree of freedom* (time).
Under this reading, time coupling is itself a derived structure:
product construction (extending X by t) followed by composition
(making other variables depend on t).

**Open question on minimality:**

Is Composition reducible to the others? In category-theoretic terms,
composition is the fundamental morphism — it may be irreducible.
Alternatively, if the state space is sufficiently structured, composition
may follow from product construction and projection.
This is the core minimality question and connects directly to Q-new-B:
whether dynamic BCs can be generated as long-time limits of static ones
depends on whether time coupling is primitive or derived.

### Sufficiency check against known BC classes

| BC class | Operator chain | Primitives required |
|---|---|---|
| Restriction | projection | projection |
| Coupling | product + composition | product, composition |
| Aggregation | projection + addition | projection, addition |
| Symmetry breaking | projection ∘ parameter variation | projection, composition |
| Forcing | time-product + multiplication + addition | product, multiplication, addition |
| Dissipation | time-product + multiplication + composition | product, multiplication, composition |

All six classes are expressible. No additional primitives appear necessary
beyond the five listed — though formal proof of sufficiency remains open.

---

# 8. Outlook

Future research directions include:

- determining minimal primitive operator sets and proving sufficiency
- constructing a formal algebra of BC compositions
- mapping BC classes to operator chains with formal derivations
- analyzing how regime structures emerge from each operator composition type
- testing the F5 failure mode empirically: identifying cases where
  collective Π failure is caused by operator basis insufficiency
  rather than partition non-existence

**Connection to Q-new-B:**

Q-new-B asks whether dynamic BCs can generate static BC structure as a
long-time limit — whether, for example, Dissipation can produce an effective
Restriction by enveloping the attractor geometry.

Under the operator framework this becomes a question about composition closure:
can `time ⊗ scaling` (Dissipation) produce a `projection` (Restriction)
as its asymptotic limit?

If the answer is yes, then static and dynamic BCs are not independent
families but related by a limiting operation — and the primitive operator
basis generates BC classes that are themselves ordered by derivational
depth, not just by scope level.

This would be the deepest unification the current framework points toward:
a single operator algebra generating all BC classes, stratified by
whether time coupling is absent, present, or taken to its limit.
